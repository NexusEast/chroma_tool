## このツールができること

Chroma Tool は**単色背景**の画像を 1 枚以上受け取り(例:緑バックの
ゲームスプライト、単色塗りつぶし背景のアイコングリッド、クロマキー
合成された商品画像など)、次の出力を生成します:

* **ピクセル単位のアルファマスク** — 前景と背景を分離しつつ、
  影と同色ディテール(葉、装飾、ドロップシャドウなど)を保持。
* **透過 PNG として切り出された画像** — 検出された各スプライトを
  1 ファイルずつ。

**バッチモード**では、同じパラメータセットを複数の入力画像に
一度に適用し、各画像の切り出し結果はその画像名で名付けられた
サブフォルダに書き出されます。

ピュアなコンピュータビジョンツールです。LLM 不使用、ネット接続不要、
テレメトリなし。すべて OpenCV と NumPy でローカル実行されます。

## 主な特徴

1. **バッチ処理。** GUI に画像を何十枚もまとめて投入(または CLI で
   フォルダを渡す)し、プレビュー画像 1 枚でパラメータを追い込み、
   ワンクリックで全画像を同じ設定で書き出せます。
2. **画像ごとの専用プロファイル。** リストで画像を切り替えると、
   その画像専用に保存されたパラメータが自動で読み込まれます。
   キーは**絶対パス**なので、同じ画像でも別の場所に置けば別の
   プロファイルになります。終了後も保存されます。
3. **自由な命名規則。** プレフィックス、開始番号、ゼロ埋め桁数を
   指定可能。プレフィックスが空ならファイル名の語幹を使用し、
   `myimage_1.png`, `myimage_2.png` ... のような連番を生成します。
4. **画像ごとの出力サブフォルダ。** 各入力画像の切り出し結果は
   その画像名のサブフォルダに分けられ、ファイル名の衝突を回避します。
5. **影を保持するキーイング。** 二重アルファパイプラインと OpenCV の
   距離変換により、影ピクセルを最も近いスプライト本体に自動で
   割り当てます。隣のスプライトに影が漏れたり、影が刈り取られたり
   しません。
6. **ソフトドロップシャドウへの再投影。** 背景色の不透明な影を
   半透明の黒(または任意の色)に変換し、どんな新しい背景に
   合成しても自然に馴染むようにできます。
7. **3 つの分割戦略。** hybrid(連結成分 + アンカー/断片マージ)、
   grid(固定セル)、contour(外輪郭)。
8. **3 つのキーイングモード。** exact(BGR 距離)、area(HSV + 大塊
   連結成分)、simple(純粋 HSV 範囲)。
9. **`_preview.png` は出力しません。** プレビュー描画は GUI 内のみで
   行われ、出力フォルダには切り出し画像のみが保存されます
   (シングル画像書き出し時にオプションで `_transparent.png` を
   含む)。
10. **3 言語 GUI** — 英語(デフォルト)、中国語、日本語。実行時に
    切り替え可能。
11. **設定の永続化。** すべてのスライダーと選択肢、画像リスト、
    最後にアクティブだった画像が起動間で復元されます。
    保存先:`%APPDATA%\ChromaTool\settings.json` および
    `profiles.json`(Linux/macOS は `~/.config/chromatool/`)。

## 向いている用途

* 単色背景の**スプライトシート**を持っていて、各スプライトを
  個別の透過 PNG に切り出したい。
* グリーンバック/ブルーバックで撮影した**商品写真の一括背景
  除去**が必要。
* CI に組み込める**スクリプト可能なバックグラウンドリムーバー**を
  ゲームアセットパイプラインで使いたい。
* デザインカンプから**アイコングリッド**を 1 個ずつのファイルに
  ばらしたい。
* インタラクティブにパラメータを追い込める **GUI ツール**で
  チューニングし、確定した設定をフォルダ全体に適用したい。

**向いていない**用途:

* 髪・毛・モーションブラーなど**ソフトエッジ**の被写体。
  これらにはアルファマッティング(例:
  [PyMatting](https://github.com/pymatting/pymatting))を後段で
  使用してください。
* **複数色の背景**や**実写の自然背景**。
  本ツールは一度に 1 色の背景しかキーアウトしません。
* **互いに接触/重なり合う**スプライト。連結成分ベースの分割では
  分離できません(`--split grid` を試すか、手作業で分離が必要)。

## インストール

```bash
pip install -r requirements.txt
```

依存:`opencv-python`、`numpy`、`Pillow`。Python 3.10 以上。

## GUI クイックスタート

```bash
python gui.py
```

1. **ファイル → 画像を追加**(または **フォルダを追加**)で左側の
   画像リストに画像を登録します。
2. リスト内の項目をクリックすると、その画像がプレビュー対象になり、
   **その画像専用のプロファイル**が自動的に読み込まれます(初回は
   現在のパラメータを継承)。
3. 右側でパラメータを調整し、**生成 / 再処理** をクリックして
   反映 — 同時に現在の画像のプロファイルが保存されます。
4. *命名と出力* パネルで **プレフィックス**、**開始番号**、
   **ゼロ埋め桁数** を設定。書き出される最初の 2 ファイル名が
   ライブプレビューされます。
5. **出力ルート** フォルダを選択。
6. **一括書き出し** をクリックすると、現在のパラメータがリスト内の
   全画像に適用され、各画像の切り出し結果はその画像名のサブフォルダに
   書き出されます。

ショートカット:`Ctrl+O` 画像追加、`Ctrl+E` 現在の画像を書き出し、
`Ctrl+B` 一括書き出し。

メニュー **言語** から `中国語 / 英語 / 日本語` を切り替えられます。

## CLI クイックスタート

シングル画像:

```bash
# (0,0) から背景色を取得し、デフォルト hybrid 分割を使用
python cli.py process input.png out_dir

# 取色するピクセルを指定
python cli.py process input.png out_dir --pick 5 5

# 背景色 BGR を直接指定
python cli.py process input.png out_dir --bg 50 180 138

# 出力命名のカスタマイズ
python cli.py process input.png out_dir \
    --name-prefix hero --name-start 0 --name-pad 3
# → hero_000.png, hero_001.png, …

# 均等スプライトシート用の固定グリッド分割
python cli.py process sheet.png out --split grid --cell-w 200 --cell-h 200
```

バッチ(各入力 → `--out-root` 配下にサブフォルダ):

```bash
python cli.py batch --out-root output/ a.png b.png c.png
python cli.py batch --out-root output/ ./sprites_folder/
python cli.py batch --out-root output/ ./sprites/ \
    --name-prefix icon --name-pad 3
```

すべてのオプションは `python cli.py process --help` または
`python cli.py batch --help` で確認できます。短縮形
`python cli.py IN OUT …`(サブコマンドなし)も受け付けられ、
`process` に振り分けられます。

## 出力ディレクトリ構造

シングル画像書き出しは選択した出力フォルダにすべて書き込みます:

```
out_dir/
├── _transparent.png       キーイング済みの全体画像(RGBA、任意)
├── myimage_1.png
├── myimage_2.png
└── …
```

バッチ書き出しは入力画像ごとにフォルダを作成します:

```
out_root/
├── myimage/
│   ├── myimage_1.png
│   ├── myimage_2.png
│   └── …
├── another/
│   ├── another_1.png
│   └── …
└── …
```

**`_preview.png` は決して出力されません。** バウンディングボックスの
プレビューは GUI 内のデバッグ表示専用です。

## 命名規則

プレフィックスを空欄にすると入力画像のファイル名語幹が使用されます。
連番は設定した開始番号から始まり、ゼロ埋めも可能です。

| プレフィックス | 開始 | ゼロ埋め | 最初の 3 ファイル                                       |
|----------|---:|---:|---------------------------------------------------|
| (空欄)   |  1 |  0 | `myimage_1.png`, `myimage_2.png`, `myimage_3.png` |
| (空欄)   |  0 |  3 | `myimage_000.png`, `myimage_001.png`, `myimage_002.png` |
| `hero`   |  1 |  2 | `hero_01.png`, `hero_02.png`, `hero_03.png`       |

## キーイングモード

| モード | 適した場面 | 動作 |
|---|---|---|
| `exact`(既定) | 単色背景(カートゥーン、UI、ゲームスプライト) | サンプリングした背景色との BGR ユークリッド距離。`d_inner` と `d_outer` の間でソフトに減衰。影と同色のインクルージョンを保持。 |
| `area` | わずかにノイジーな背景で、物体内部の同色領域を残したい場合 | HSV 範囲マスク → 連結成分 → 大きい塊のみを背景と判定。 |
| `simple` | 完全なグリーンバックで、影を保持する必要がない場合 | 純粋な HSV 範囲。最速・最も破壊的。 |

## 分割モード

| モード | 適した場面 |
|---|---|
| `hybrid`(既定) | スプライト間に背景の隙間がある場合。アンカーが断片を吸収し、二重アルファの距離変換で影ピクセルを親に再帰属させます。 |
| `grid` | セルサイズが既知の均等スプライトシート。 |
| `contour` | 軽く重なるアイコン。外輪郭ポリゴンを使用。 |
| `none` | 背景除去のみで分割は行いません。 |

## 影の処理

| モード | 新背景に合成した結果 |
|---|---|
| `soft`(既定) | 各スプライトの下に半透明のドロップシャドウ — どんな新しい背景にも自然に馴染みます。 |
| `keep` | 元の不透明な背景色の影をそのまま残します。 |
| `remove` | スプライトが浮いた状態(影なし)。 |

調整パラメータ:`--shadow-intensity`(0.5 – 3.0、大きいほど濃い)、
`--shadow-max-alpha`(0 – 255 の上限)、`--shadow-color`(影の色合い
BGR)。

## チューニングチートシート

| 現象 | 調整 |
|---|---|
| 背景が残ってしまう | `--d-outer` を上げる(HSV モードでは `--hue-tol`) |
| 影や同色ディテールが消える | `--d-outer` を下げ、`--d-inner` を上げる |
| 影が濃すぎ/薄すぎ | `--shadow-intensity` と `--shadow-max-alpha` を調整 |
| 隣接スプライトが 1 つに連結されてしまう | `--bridge-erode` を上げ、`--shadow-distance` を下げ、`--strict-d-inner` を上げる |
| 細かい断片が大量に出る | `--min-area` を上げる |
| 装飾が親から分離する | `--merge-distance` または `--shadow-distance` を上げる |
| 物体の周囲に色のハロー | `--feather` を上げ、デコンタミネーションは有効のまま |

## プロジェクト構成

```
chroma_tool/
├── io_utils.py     Unicode 安全な画像読み書き + ディレクトリ走査
├── keying.py       背景検出(exact / area / simple)
├── shadows.py      影のソフト化 / 保持 / 削除
├── splitting.py    hybrid / grid / contour の分割戦略
├── pipeline.py     ProcessParams + process_image + export_crops
├── batch.py        複数画像のバッチランナー(画像ごとの結果付き)
├── naming.py       NamingPattern とファイル名生成
├── profiles.py     画像パスごとの設定プロファイル(永続化)
├── settings.py     グローバル GUI 設定(言語、命名、画像リスト)
├── i18n.py         GUI の中・英・日翻訳
├── cli.py          コマンドライン入口
├── gui.py          Tkinter フロントエンド
├── requirements.txt
├── README.md       (英語)
├── README.zh.md    (中国語)
└── README.ja.md    (このファイル)
```

各レイヤーは独自の凍結 `dataclass` でパラメータを持ち、
`pipeline.ProcessParams` がそれらを組み合わせます。
`cli.py` と `gui.py` はどちらも `ProcessParams` を構築して
`pipeline.process_image`(シングル画像)または
`batch.batch_process`(複数画像)に渡すだけで、ビジネスロジックは
フロントエンドに置かれていません。

## FAQ

**Q: ネット接続や LLM が必要ですか?**
A: いいえ。完全オフラインのコンピュータビジョン(OpenCV + NumPy)
のみです。

**Q: Python から直接呼び出せますか?**
A: はい。`pipeline.process_image` はシングル画像、
`batch.batch_process` は複数画像用です。どちらも 1 つの
`ProcessParams` dataclass を受け取ります。

**Q: サーバや CI でヘッドレス実行できますか?**
A: `cli.py` を使ってください。GUI は任意です。

**Q: 各スプライトの下の影は保持されますか?**
A: はい — 二重アルファ `exact` パイプラインの目的そのものです。
`shadow_mode=soft` を選ぶと、新しい背景に合成しても自然な
ドロップシャドウになります。

**Q: Windows で非 ASCII のファイル名は使えますか?**
A: はい。画像 I/O はすべて `numpy.fromfile` / `cv2.imencode` を
経由しており、`cv2.imread` / `cv2.imwrite` が失敗する Unicode パスでも
正しく動作します。

**Q: スプライトが互いに接触しています。どうすれば?**
A: 連結成分では分離できません。`--bridge-erode 2` で薄いピクセル橋を
切断するか、`--strict-d-inner` を上げて本体マスクを締めてみてください。
均等グリッドなら `--split grid` への切り替えが確実です。

**Q: `_preview.png` は出力されますか?**
A: いいえ。プレビュー描画は GUI 専用で、出力フォルダには切り出し
画像のみが含まれます(シングル画像書き出し時のオプションの
`_transparent.png` を除く)。

**Q: 画像ごとの設定はどこに保存されますか?**
A: Windows: `%APPDATA%\ChromaTool\profiles.json`。
   Linux/macOS: `~/.config/chromatool/profiles.json`。
画像パス(絶対パスを正規化したもの)をキーとした JSON です。
同じ画像でも別の場所に置けば別プロファイルになります。

**Q: 他の言語のドキュメントはありますか?**
A: あります。
[English README](README.md) と [中文 README](README.zh.md) を
参照してください。

## 既知の制限

* **連結成分では接触するスプライトを分離できません。** 1 ピクセルでも
  前景を共有する 2 つのスプライトは 1 つとして切り出されます。
  `--bridge-erode` を試すか、均等シートなら `--split grid` を
  使用してください。
* **1 画像につき 1 背景色のみ。** 複数色の背景には複数回のキーイング
  パスが必要です(本ツールには未実装)。
* **髪/毛のアルファマッティングは未対応。** 非常にソフトなエッジには
  本ツールが生成したアルファに対して
  [PyMatting](https://github.com/pymatting/pymatting) を後段で
  適用してください。
